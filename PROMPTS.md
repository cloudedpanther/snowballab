# Stage 진행 프롬프트 템플릿

## 필수 규칙(최상단 고정)

- 한 번에 한 Stage만 수행한다.
- 해당 Stage 범위를 절대 넘지 않는다.
- 불확실한 부분은 기본값으로 결정하고 TODO로 남긴다.
- npm만 사용한다(pnpm/yarn 금지).
- 작업 끝에 변경 파일 목록/변경 이유/검사 명령어/커밋 메시지 예시를 항상 포함한다.

---

## 복붙용 템플릿

아래 Stage만 수행해줘. 범위 밖 작업/리팩토링/최적화 금지.

[Stage]

- Stage: {번호}
- 목표: {Stage 목표}

[작업 범위 제한]

- 포함: {이번 Stage에 포함되는 작업}
- 제외: {이번 Stage에서 하면 안 되는 작업}

[변경 파일]

- 변경 파일 목록과 변경 이유를 반드시 요약해줘.

[검사 방법]

- 내가 터미널에서 실행할 명령어를 제공해줘(npm 기준).
- 예: `npm install`, `npm run dev`, `npm run build`, `npm run format`, `npm run check`

[TODO 규칙]

- TODO는 불확실하거나 나중에 해야 할 항목만.
- TODO는 짧고 명확하게.

[출력 형식]

1. 작업 내용 요약
2. 변경 파일 목록 + 변경 이유
3. 검사 명령어
4. 커밋 메시지 예시
5. 남은 TODO(있다면)
